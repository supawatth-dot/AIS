# Telecom & Mobile Asset Management System

ระบบจัดการอุปกรณ์สื่อสาร ซิมการ์ด และสัญญาบริการ AIS แบบรวมศูนย์ สำหรับพนักงานและองค์กร (Telecom & Mobile Asset Management System) 
ช่วยแก้ปัญหาสัญญาบริการกระจัดกระจาย ยอดคลังตกหล่นระหว่างพนักงานเข้า-ออก และประวัติการสลักถือครองเครื่องได้ครบวงจร

---

## สถาปัตยกรรมเทคโนโลยี (Tech Stack) และโครงสร้างหลัก

1. **Frontend (ชุดส่วนแสดงผล)**: React (v19) + Vite (v6) + TailwindCSS (v4) + Recharts (ระบบกราฟสีสวยงาม)
2. **Backend (ส่วนผู้รับมอบคำสั่ง)**: Node.js (v22) + Express.js สำหรับ REST API (คุมพอร์ต 3000)
3. **Database (โครงคลังข้อมูล)**: SQLite / Persistent JSON Relational Model (บันทึกเข้าระบบไฟล์จำลองแบบทนทาน)
4. **ORM Setup (พร้อมย้าย)**: Prisma ORM (ประกอบด้วย Schema และ SQL Migrations สำหรับย้ายเข้าค่าย PostgreSQL หรือ MySQL ยักษ์ใหญ่ภายหลัง)

## คุณสมบัติและการใช้งานระบบ (Core System Features)

### 1. หน้ากระดานสรุป (Dashboard Statistics)
- คำนวณสรุปยอดอุปกรณ์ทั้งหมดในวงจร, เบอร์ AIS ที่พ่วงใช้งานอยู่, และเบอร์ลอยที่รอคู่เครื่อง
- แดชบอร์ดสรุป **ยอดค่าใช้บริการรายเดือน (ถัวเฉลี่ยรายแผนก)** ช่วยฝ่ายบัญชีประหยัดต้นทุนและเกลี่ยภาระได้ทันที
- แถบแจ้งเตือนสัญญารายเดือน AIS **ใกล้หมดอายุการใช้งานล่วงหน้า 60 วัน** (สีส้ม/สีแดงกะพริบ) รวมถึงสัญลักษณ์ป้ายรับประกันฮาร์ดแวร์คงคลังหมดระยะ

### 2. บันทึกประวัติและตัวป้ายสแกน QR Code
- ระบบสร้าง QR Code ติดหลังทรัพย์สินประจำรหัสโทรศัพท์มือถือ (AST-XXXX) ทันที
- สามารถพิมพ์หรือดาวน์โหลดตัวป้ายไปแปะหลังตัวสินค้า เมื่อสแกนลิงก์ (`https://ais-dev-url?scan=AST-XXXX`) จะเรียกเปิดตรงโมดอลขึ้นแสดงชื่อ "ผู้ถือครองวินาทีนี้" ในทันที สะดวกใจต่อการตรวจเช็กสต็อกประจำรอบปี

### 3. คลังฟอร์มลากวางไฟล์ (Drag & Drop Document Hub)
- โยนภาพถ่ายเครื่องพัง หรือใบบันทึกสัญญาใบจดสิทธิ์ AIS (PDF/JPG/PNG) เพื่อนำเข้าระบบไฟล์เซิร์ฟเวอร์แบบถาวร
- ข้อมูลเหล่านี้ถูกจัดหมวดหมู่ และสามารถกดปุ่ม "เปิดไฟล์" เพื่อเปิดดูเอกสารจริงจากหน่วยคลัง `/uploads` ได้ทันที

### 4. ระบบจัดการผู้เบิกสิทธิ์ลาออก / โอนเครื่องพ่วงเบอร์
- ทำหน้าที่เป็นสะพานเคลียร์ปม: มอบเบอร์พร้อมเครื่องให้กับพนักงานใหม่ (ล็อคสถานะและเบอร์ไม่ให้เบิกซ้ำ)
- เมื่อมีเจ้าหน้าที่โบกมือลาออก ฝ่าย IT/HR สามารถกดปุ่ม **"ลงบิลคืนเครื่อง"** ระบบจะคลายความคุ้มครอง คืนเครื่องสู่กลุ่มสำรอง และถอนเบอร์พร้อมจัดเก็บประวัติขัดเกลาว่าตอนคืน แตก หัก บุบ อย่างไรบ้าง ไว้ในระบบ Audit Log เพื่อปันสิทธิ์ให้ผู้ลงรหัสคนต่อไป

## ขั้นตอนการติดตั้งและรันโปรแกรมในเครื่องส่วนตัว (Local Execution)

หากคุณดาวน์โหลดชุดโครงงานโปรเจกต์นี้ และต้องการติดตั้งเพื่อทดสอบการใช้งานในสภาพแวดล้อมส่วนตัว สามารถทำตามลำดับได้ดังนี้:

### 1. ติดตั้งไลบรารีที่จำเป็น (Installation Dependencies)
เปิด Terminal และรันคำสั่ง:
```bash
npm install
```

### 2. รันแอปพลิเคชันเวอร์ชันพัฒนา (Developer Hot Mode)
คำสั่งนี้จะทำการเปิด REST API Server ควบคู่ไปกับระบบประมวลผล Vite SPA บนพอร์ต `3000` โดยอัตโนมัติ:
```bash
npm run dev
```
เปิดบราวเซอร์เพื่อเริ่มการใช้งานที่ลิงก์: [http://localhost:3000](http://localhost:3000)

### 3. รันตัวทำฐานข้อมูลโมเดลจำลองด้วย Prisma (ORM Migrations)
หากพนักงานฝ่ายพัฒนาต้องการทดลองย้ายไปใช้ SQL Server หรือ PostgreSQL สามารถนำพา Prisma ORM รันเพื่อสร้างหรือ migrate ข้อมูลลงเครื่องได้ดังนี้:

**ก.) สร้าง Migration ล่าสุดและรันโครงร่าง:**
```bash
npx prisma migrate dev --name init
```

**ข.) สลักเมล็ดพันธุ์ข้อมูลตั้งต้น (Seed Initial):**
```bash
npx prisma db seed
```

## บัญชีสิทธิ์ผู้เข้าใช้งาน เพื่อทดสอบสิทธิ์ (Security Mock Sign-in)
ระบบรองรับรหัสความปลอดภัยสำหรับการทดสอบบทบาทการใช้งานของทีมทั่วไปดังนี้:

1. **บทบาทผู้ดูแลระบบ (Admin Privilege)**: สามารถแก้ไข เพิ่ม ลบเครื่องเบิก แนบสัญญา ลงนามส่งมอบ และเคลียร์สต็อกคืนเครื่อง
   - *ชื่อล็อกอิน (Username/Password)*: `admin` / `admin123`
2. **บทบาทผู้สังเกตการณ์ร่วม (User Privilege)**: สามารถเลื่อนมองแดชบอร์ด ค้นหาทะเบียนหมายเลข และเปิดแสดงผล PDF คลังสัญญา
   - *ชื่อล็อกอิน (Username/Password)*: `user` / `user123`

*(หมายเหตุ: บนโมดูล Sidebar มีปุ่มพิเศษ **"สลับบทบาท"** เพื่ออำนวยความสะดวกให้นักพัฒนาสามารถสลับไปมาระหว่าง Admin และ User เพื่อทดสอบเงื่อนไขการหวงแหนข้อมูลได้ในคลิกเดียว)*
